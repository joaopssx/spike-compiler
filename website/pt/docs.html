Arquitetura

Como o Spike e dividido internamente.

Cada modulo do projeto tem uma responsabilidade bem definida, o que deixa o compilador mais facil de manter e de estudar.

Lexer

Le o codigo fonte caractere por caractere e produz um `std::vector` com localizacao precisa.

Parser

Usa recursive descent para transformar tokens em uma AST com precedencia correta.

AST

Representa declaracoes, comandos e expressoes em memoria usando `std::unique_ptr`.

Analise Semantica

Valida simbolos, atribuicoes, tipos e condicoes antes que o C seja gerado.

Code Generator

Percorre a AST e emite um arquivo C valido e legivel.

CLI

Costura o pipeline inteiro e expoe modos de build, run, tokens, ast e emit-c.

MVP

Subconjunto atual da linguagem.

CLI

Comandos mais importantes.

Compilar

spike build examples/hello.por

Gera o `.c` e o binario final.

Executar

spike run examples/hello.por

Compila e executa o binario gerado.

Inspecionar

spike tokens arquivo.por
spike ast arquivo.por

Ajuda a entender como o compilador esta lendo o programa.

Roadmap

O que vem depois do MVP.
