Tutorial guiado

Do zero ao primeiro executavel com Spike.

Este tutorial percorre o fluxo real do compilador: build do projeto, criacao de um programa em Portugol, inspecao de tokens, visualizacao da AST, geracao de C e execucao do binario final.

Passo 1

Faca o build do Spike.

Comece compilando o proprio compilador com CMake. Esse passo funciona tanto para Linux e macOS quanto para Windows com geradores compativeis.

Comandos

build local 
cmake -S . -B build
cmake --build build

Resultado esperado

Ao final, voce deve ter o executavel `build/spike`. Esse binario e o front-end do compilador e tambem o ponto de entrada da CLI.

Dica: se quiser checar a CLI primeiro, rode `./build/spike --help`.
Passo 2

Crie o seu primeiro programa em Portugol.

Fonte `hello.por`

hello.por 
algoritmo "hello"

var
    a: inteiro

inicio
    a <- 10
    escreva(a)
fimalgoritmo

O que esse programa faz

Ele declara uma variavel `a`, atribui o valor `10` e imprime esse valor. E o menor exemplo util para verificar se o pipeline inteiro do compilador esta funcionando.

Passo 3

Inspecione os tokens.

Antes do parser, vale garantir que o lexer esta lendo o codigo corretamente.

Comando

tokens 
./build/spike tokens hello.por

Saida esperada

saida 
Algoritmo -> algoritmo
String -> hello
Var -> var
Identifier -> a
Colon -> :
Inteiro -> inteiro
Inicio -> inicio
Identifier -> a
Assign -> <-
NumberInt -> 10
Escreva -> escreva
LParen -> (
Identifier -> a
RParen -> )
FimAlgoritmo -> fimalgoritmo
EOF
Passo 4

Visualize a AST.

Depois de tokenizar, o parser transforma a entrada em uma arvore interna. Esse passo ajuda a confirmar se a estrutura do programa foi entendida corretamente.

Comando

ast 
./build/spike ast hello.por

Saida esperada

ast 
Programa("hello")
  Declaracoes
    Declaracao(inteiro: a)
  Comandos
    Atribuicao(a)
      Inteiro(10)
    Escreva
      Variavel(a)
Passo 5

Gere e leia o codigo C.

O Spike transpila Portugol para C antes de chamar o compilador nativo. Esse arquivo intermediario e excelente para aprendizado e debug.

Comando

emit-c 
./build/spike emit-c hello.por --emit-c generated/hello.c

Resultado tipico

generated/hello.c 
#include <stdio.h>

int main(void) {
    int a;

    a = 10;
    printf("%d\n", a);

    return 0;
}
Passo 6

Compile e execute o programa final.

Comando final

run 
./build/spike run hello.por

Fluxo interno

saida 
Arquivo C gerado em generated/hello.c
Binario gerado em generated/hello
10
Passo 7

Use o pipeline para depurar e crescer o projeto.

Quando usar `tokens`

Quando o problema parece lexical: simbolos estranhos, palavras-chave nao reconhecidas ou operadores quebrados.

Quando usar `ast`

Quando a estrutura do programa parece errada ou quando uma expressao nao esta respeitando a precedencia correta.

Quando usar `emit-c`

Quando voce quer comparar a AST com o C emitido e validar se o codegen esta fiel ao programa original.

Quando usar `run`

Quando o pipeline ja parece consistente e voce quer validar o comportamento final do executavel nativo.

Proximos passos

Agora experimente exemplos maiores.

Depois do `hello`, vale rodar `examples/if.por`, `examples/loop.por` e `examples/fibonacci.por` para ver condicoes, lacos e expressoes maiores em acao.
