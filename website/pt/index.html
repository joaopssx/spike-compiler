Portugol moderno, pipeline claro, executavel nativo.

Lexer

Quebra o fonte em tokens com linha e coluna.

Parser

Semantica

Codegen

Pipeline visual

Da linguagem didatica ao executavel real.

O fluxo do Spike e intencionalmente transparente para facilitar estudo, debug e evolucao do compilador.

1. Source

Le o arquivo `.por` do usuario.

2. Lexer

Transforma texto em tokens nomeados.

3. Parser

Converte os tokens em AST.

4. Semantica

Checa variaveis, tipos e expressoes.

5. C

Gera um arquivo `.c` intermediario.

6. Binario

Usa o compilador C para criar o executavel.

Por que Spike?

Um projeto bonito para estudar compiladores de verdade.

Didatico sem ser simplista

O codigo privilegia clareza, mas ja usa uma arquitetura real de compilador com AST, semantica e codegen separados.

Moderno em C++

`std::unique_ptr`, `std::vector`, `std::unordered_map` e CMake deixam o projeto organizado e portavel para Linux, macOS e Windows.

Open source pronto

O repositorio ja tem exemplos, docs, CI, site estatico e deploy pensado para GitHub Pages.

CLI

Comandos diretos e profissionais.

O Spike expoe uma CLI simples para compilar, executar e depurar o pipeline por etapas.

Build

spike build examples/hello.por

Run

spike run examples/hello.por

Debug

spike tokens examples/hello.por
spike ast examples/if.por
GitHub Pages

Site pronto para publicar.

O site estatico do Spike foi pensado para rodar direto no GitHub Pages, com deploy automatico por GitHub Actions e links relativos entre as paginas.

Paginas

Inicio, docs e tutorial com layout responsivo.

Deploy

Um workflow dedicado publica o conteudo de `website/` automaticamente.

Use `Settings -> Pages -> Build and deployment -> GitHub Actions` no repositorio.
