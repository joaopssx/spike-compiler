Architecture

How Spike is divided internally.

Each module has a clear responsibility, which keeps the compiler easier to maintain and easier to study.

Lexer

Reads the source character by character and produces a `std::vector` with accurate locations.

Parser

Uses recursive descent to turn tokens into an AST with correct precedence.

AST

Represents declarations, statements and expressions in memory with `std::unique_ptr`.

Semantic Analysis

Validates symbols, assignments, types and conditions before C output exists.

Code Generator

Walks the AST and emits valid, readable C source code.

CLI

Connects the whole pipeline and exposes build, run, tokens, ast and emit-c modes.

MVP

The currently supported language subset.

Supported structures

algoritmo var inicio fimalgoritmo se / senao / fimse enquanto / faca / fimenquanto escreva leia

Types and operators

inteiro real <- + - * / = <> < <= > >=
CLI

Most important commands.

Build

spike build examples/hello.por

Generates the `.c` file and the final binary.

Run

spike run examples/hello.por

Builds and executes the generated binary.

Inspect

spike tokens file.por
spike ast file.por

Helps you understand how the compiler is reading the program.

Roadmap

What comes after the MVP.

The current plan includes better diagnostics, stronger test coverage, expanded type support and future ideas such as an interpreter, formatter and possibly an LLVM backend.
